## Overview

ARMS uses a centralized parameter system to store configurable values that control business logic, thresholds, and defaults. Parameters are stored in the `parameter` database table and accessed via the `getParameter` function with built-in caching.

## Parameter table structure

| Column | Type | Description |
|--------|------|-------------|
| `param_key` | string | Unique identifier for the parameter (e.g., `FORFAIT_1_DAY_EUR`) |
| `param_value` | string | The stored value as a string (cast at runtime) |
| `param_type` | enum | Type for casting: `integer`, `decimal`, `boolean`, `text`, `email_list` |
| `description_nl` | string | Dutch description of the parameter's purpose |
| `description_fr` | string | French description of the parameter's purpose |

## Type casting

All parameter values are stored as strings and cast to their target type at read time using `castParameterValue`:

```typescript parameters.ts
export type ParamType = "integer" | "decimal" | "boolean" | "text" | "email_list";
export type ParameterValue = number | boolean | string | string[];
```

| Type | Cast logic | Example input | Output |
|------|------------|---------------|--------|
| `integer` | `parseInt(value, 10)` | `"14"` | `14` |
| `decimal` | `parseFloat(value)` | `"0.30"` | `0.3` |
| `boolean` | `"true"` or `"1"` = true, else false | `"true"` | `true` |
| `text` | No transformation | `"hello"` | `"hello"` |
| `email_list` | Split by comma, trim whitespace | `"a@b.com, c@d.com"` | `["a@b.com", "c@d.com"]` |

<Callout kind="alert">
  If a value cannot be cast to `integer` or `decimal`, the function throws an error. Ensure parameter values are valid before saving to the database.
</Callout>

## Caching strategy

Parameters are cached using Next.js `unstable_cache` to minimize database queries:

```typescript parameters.ts
const getCachedParameter = unstable_cache(
  async (key: string) => fetchParameter(key),
  ["parameter"],
  { revalidate: 300, tags: ["parameters"] }
);
```

| Setting | Value | Description |
|---------|-------|-------------|
| Cache key prefix | `"parameter"` | Shared namespace for all parameters |
| Revalidation interval | 300 seconds (5 minutes) | How often the cache refreshes from the database |
| Cache tag | `"parameters"` | Used for manual cache invalidation via `revalidateTag("parameters")` |

<Callout kind="tip">
  After updating a parameter value in the admin UI, the change takes effect within 5 minutes due to the cache revalidation interval. For immediate effect, trigger a manual cache revalidation.
</Callout>

## API reference

### getParameter

Returns the cast value for a parameter key. Returns `null` if the key does not exist.

```typescript
import { getParameter } from "@/lib/parameters";

const forfait1Day = await getParameter("FORFAIT_1_DAY_EUR");
// Returns: 120 (as number)

const alertRecipients = await getParameter("INSPECTION_ALERT_RECIPIENTS");
// Returns: ["user@example.com", "admin@example.com"] (as string[])

const unknown = await getParameter("NONEXISTENT_KEY");
// Returns: null
```

### getParameterRaw

Returns the full `ParameterRow` object without type casting. Useful for admin interfaces that need to display the raw value and type.

```typescript
import { getParameterRaw } from "@/lib/parameters";

const raw = await getParameterRaw("FORFAIT_1_DAY_EUR");
// Returns: {
//   param_key: "FORFAIT_1_DAY_EUR",
//   param_value: "120",
//   param_type: "decimal",
//   description_nl: "Forfaitbedrag voor contractduur van 1 dag",
//   description_fr: "Montant forfaitaire pour une durée de contrat de 1 jour"
// }
```

### castParameterValue

Pure function that casts a string value to the target type. Used internally by `getParameter` but also available for direct use.

```typescript
import { castParameterValue } from "@/lib/parameters";

castParameterValue("120", "decimal");     // 120
castParameterValue("true", "boolean");    // true
castParameterValue("14", "integer");      // 14
castParameterValue("a@b.com,c@d.com", "email_list"); // ["a@b.com", "c@d.com"]
```

## Known parameters

The following parameters are seeded in the database and used throughout ARMS:

### Inspection parameters

| Key | Default | Type | Description |
|-----|---------|------|-------------|
| `INSPECTION_WARNING_DAYS` | `14` | integer | Days before inspection expiry to send a notification |
| `INSPECTION_ALERT_RECIPIENTS` | (empty) | email_list | Comma-separated email recipients for inspection alerts |

### Invoicing parameters

| Key | Default | Type | Description |
|-----|---------|------|-------------|
| `FORFAIT_1_DAY_EUR` | `120` | decimal | Flat-rate amount for 1-day contracts |
| `FORFAIT_2_DAYS_EUR` | `170` | decimal | Flat-rate amount for 2-day contracts |
| `KM_YEARLY_MAX` | `100000` | integer | Maximum km per year included in contract |
| `KM_SURPLUS_PRICE_EUR` | `0.30` | decimal | Price per km above the yearly maximum |
| `DEPOSIT_DEFAULT_AMOUNT_EUR` | `2000` | decimal | Default deposit amount for new contracts |

### Contract parameters

| Key | Default | Type | Description |
|-----|---------|------|-------------|
| `OFFER_DUE_DATE_OFFSET_WEEKS` | `1` | integer | Default offset in weeks for offer due date |
| `NON_DRIVING_KM_THRESHOLD` | `10` | integer | Km/day threshold for red marking in non-driving periods |
| `CONTRACT_FOLLOWUP_WARNING_DAYS` | `7` | integer | Days after sending before contract turns yellow (no response) |

### Session and security parameters

| Key | Default | Type | Description |
|-----|---------|------|-------------|
| `SESSION_TIMEOUT_MINUTES` | `60` | integer | Inactivity timeout for automatic logout |

### Document settings

| Key | Default | Type | Description |
|-----|---------|------|-------------|
| `MAX_UPLOAD_SIZE_MB` | `25` | integer | Maximum file size per upload in megabytes |

## Adding new parameters

To add a new parameter:

1. Insert a row into the `parameter` table with the appropriate key, value, type, and descriptions
2. Access it in application code using `getParameter("YOUR_KEY")`
3. The value will be cached and type-cast automatically

```sql
INSERT INTO parameter (param_key, param_value, param_type, description_nl, description_fr)
VALUES (
  'NEW_PARAMETER_KEY',
  '100',
  'integer',
  'Nederlandse beschrijving',
  'Description en francais'
);
```
