## Overview

ARMS supports two locales: **Dutch (NL)** as the primary language and **French (FR)** as the secondary language. The internationalization is powered by next-intl, which provides server-side message loading, locale detection via cookies, and formatted dates/numbers/currencies.

## Configuration

### Locale setup (`i18n/config.ts`)

```typescript
export const locales = ["nl", "fr"] as const;
export type Locale = (typeof locales)[number];
export const defaultLocale: Locale = "nl";

export const localeLabels: Record<Locale, string> = {
  nl: "Nederlands",
  fr: "Francais",
};

export const LOCALE_COOKIE = "locale";
```

### Request configuration (`i18n/request.ts`)

next-intl is configured as a Next.js plugin in `next.config.ts`:

```typescript
import createNextIntlPlugin from "next-intl/plugin";

const withNextIntl = createNextIntlPlugin("./i18n/request.ts");
export default withNextIntl(nextConfig);
```

The request configuration reads the locale from a cookie and loads the corresponding message file:

```typescript
export default getRequestConfig(async () => {
  const store = await cookies();
  const raw = store.get(LOCALE_COOKIE)?.value;
  const locale: Locale = locales.includes(raw as Locale)
    ? (raw as Locale)
    : defaultLocale;

  const messages = (await import(`../messages/${locale}.json`)).default;

  return { locale, messages, formats: { ... } };
});
```

## Locale detection

ARMS uses cookie-based locale detection rather than URL-based routing:

1. Check the `locale` cookie value
2. If the cookie value is a valid locale (`"nl"` or `"fr"`), use it
3. Otherwise, fall back to the default locale (`"nl"`)

Users switch languages using the `LanguageSwitcher` component in the UI, which sets the cookie.

## Message files

### Structure

Messages are organized by namespace in JSON files:

```
messages/
  nl.json    # Dutch translations
  fr.json    # French translations
```

**Namespace examples from `nl.json`:**

```json
{
  "common": {
    "appName": "ARMS",
    "save": "Opslaan",
    "cancel": "Annuleren",
    "delete": "Verwijderen",
    "search": "Zoeken",
    "loading": "Laden..."
  },
  "nav": {
    "dashboard": "Dashboard",
    "fleet": "Vloot",
    "customers": "Klanten",
    "offers": "Offertes",
    "contracts": "Contracten",
    "invoices": "Facturatie"
  },
  "validation": {
    "required": "Dit veld is verplicht",
    "invalidEmail": "Ongeldig e-mailadres",
    "minLength": "Minimaal {min} tekens vereist"
  }
}
```

### Using translations in components

```typescript
import { useTranslations } from "next-intl";

function MyComponent() {
  const t = useTranslations("common");

  return <button>{t("save")}</button>;
  // Renders: "Opslaan" (NL) or "Enregistrer" (FR)
}
```

## Formatted values

The i18n configuration includes predefined formats for dates, times, numbers, and currency:

### Date and time formats

| Format | Pattern | Example (NL) |
|--------|---------|-------------|
| `short` | DD/MM/YYYY | 04/03/2026 |
| `long` | weekday, day month year | woensdag 4 maart 2026 |
| `time` | HH:MM | 14:30 |

### Number formats

| Format | Style | Example |
|--------|-------|---------|
| `currency` | EUR currency | EUR 1.250,00 |
| `decimal` | 2 decimal places | 1.250,50 |
| `integer` | No decimals | 1.250 |

## Bilingual dropdown values

Database dropdown values (trailer types, statuses, payment conditions, etc.) are stored with both `value_nl` and `value_fr` columns. The UI renders the appropriate value based on the active locale:

```typescript
// Dropdown values from the database
{
  value_id: 1,
  value_nl: "Actief",
  value_fr: "Actif"
}
```

The status badges, dropdown menus, and table displays select the correct language column dynamically.

## Adding new translations

<Steps>
  <Step title="Add the key to nl.json" icon="file-text" titleType="p">
    Add your translation key in the appropriate namespace:

    ```json
    {
      "fleet": {
        "newTrailer": "Nieuwe trailer"
      }
    }
    ```
  </Step>

  <Step title="Add the corresponding key to fr.json" icon="file-text" titleType="p">
    ```json
    {
      "fleet": {
        "newTrailer": "Nouvelle remorque"
      }
    }
    ```
  </Step>

  <Step title="Use the translation in your component" icon="code" titleType="p">
    ```typescript
    const t = useTranslations("fleet");
    return <h1>{t("newTrailer")}</h1>;
    ```
  </Step>
</Steps>

<Callout kind="alert">
  Always add translations to both `nl.json` and `fr.json`. Missing keys will cause runtime errors when the user switches to a locale where the key is not defined.
</Callout>
