## Overview

The core tables provide shared infrastructure for the entire ARMS application. These tables store company information, physical locations, centralized dropdown values, system parameters, and email notification history.

All core tables are readable by any authenticated user. Only users with the `admin` role can create, update, or delete records.

## company

Legal entities that own trailers and issue rental contracts. Each company maintains its own invoice number sequence.

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| `company_id` | bigint (identity PK) | Yes | Auto-generated unique identifier |
| `company_name` | text | Yes | Display name of the company |
| `vat_number` | text | Yes | VAT registration number |
| `street` | text | Yes | Street name |
| `number` | text | Yes | Street number |
| `postal_code` | text | Yes | Postal code |
| `city` | text | Yes | City name |
| `country` | text | Yes | Country name |
| `default_offer_template_nl` | text | No | Default Dutch offer template reference |
| `default_offer_template_fr` | text | No | Default French offer template reference |
| `default_contract_template_nl` | text | No | Default Dutch contract template reference |
| `default_contract_template_fr` | text | No | Default French contract template reference |
| `created_on` | timestamptz | Yes | Record creation timestamp |
| `created_by` | uuid | No | User who created the record |
| `updated_on` | timestamptz | Yes | Last update timestamp |
| `updated_by` | uuid | No | User who last updated the record |

<Expandable title="SQL definition" default-open="false">
```sql
create table company (
  company_id   bigint generated always as identity primary key,
  company_name text not null,
  vat_number   text not null,
  street       text not null,
  number       text not null,
  postal_code  text not null,
  city         text not null,
  country      text not null,

  default_offer_template_nl    text,
  default_offer_template_fr    text,
  default_contract_template_nl text,
  default_contract_template_fr text,

  created_on timestamptz not null default now(),
  created_by uuid references auth.users(id),
  updated_on timestamptz not null default now(),
  updated_by uuid references auth.users(id)
);
```
</Expandable>

## location

Physical locations belonging to a company. Trailers are assigned to locations for organizational tracking.

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| `location_id` | bigint (identity PK) | Yes | Auto-generated unique identifier |
| `company_id` | bigint (FK) | Yes | Parent company reference |
| `location_name` | text | Yes | Display name of the location |
| `street` | text | Yes | Street name |
| `number` | text | Yes | Street number |
| `postal_code` | text | Yes | Postal code |
| `city` | text | Yes | City name |
| `country` | text | Yes | Country name |
| `created_on` | timestamptz | Yes | Record creation timestamp |
| `created_by` | uuid | No | User who created the record |
| `updated_on` | timestamptz | Yes | Last update timestamp |
| `updated_by` | uuid | No | User who last updated the record |

**Indexes:**

| Index | Columns | Purpose |
|-------|---------|---------|
| `idx_location_company_id` | `company_id` | Fast lookup of locations by company |

<Expandable title="SQL definition" default-open="false">
```sql
create table location (
  location_id   bigint generated always as identity primary key,
  company_id    bigint not null references company(company_id),
  location_name text not null,
  street        text not null,
  number        text not null,
  postal_code   text not null,
  city          text not null,
  country       text not null,

  created_on timestamptz not null default now(),
  created_by uuid references auth.users(id),
  updated_on timestamptz not null default now(),
  updated_by uuid references auth.users(id)
);

create index idx_location_company_id on location(company_id);
```
</Expandable>

## dropdown_value

Centralized lookup table for all dropdown menus in the application. Values are grouped by `category` and support bilingual labels (Dutch and French).

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| `value_id` | bigint (identity PK) | Yes | Auto-generated unique identifier |
| `category` | text | Yes | Groups values by purpose (e.g., `trailer_status`, `offer_status`) |
| `value_nl` | text | Yes | Dutch label |
| `value_fr` | text | Yes | French label |
| `is_active` | boolean | Yes | Whether this value appears in dropdowns (default: `true`) |
| `sort_order` | integer | Yes | Display order within the category (default: `0`) |
| `created_on` | timestamptz | Yes | Record creation timestamp |
| `created_by` | uuid | No | User who created the record |
| `updated_on` | timestamptz | Yes | Last update timestamp |
| `updated_by` | uuid | No | User who last updated the record |

**Indexes:**

| Index | Columns | Purpose |
|-------|---------|---------|
| `idx_dropdown_value_category` | `category` | Fast filtering by category |

<Callout kind="info">
  The `dropdown_value` table is referenced by nearly every domain table in the schema. Categories include trailer types, statuses, sheet types, models, door types, payment conditions, and more. Deactivating a value (`is_active = false`) hides it from dropdowns without breaking existing references.
</Callout>

<Expandable title="SQL definition" default-open="false">
```sql
create table dropdown_value (
  value_id   bigint generated always as identity primary key,
  category   text not null,
  value_nl   text not null,
  value_fr   text not null,
  is_active  boolean not null default true,
  sort_order integer not null default 0,

  created_on timestamptz not null default now(),
  created_by uuid references auth.users(id),
  updated_on timestamptz not null default now(),
  updated_by uuid references auth.users(id)
);

create index idx_dropdown_value_category on dropdown_value(category);
```
</Expandable>

## parameter

System-wide configuration parameters stored as key-value pairs with typed validation. Used for settings such as automatic logout timeout, email recipient lists, and other configurable values.

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| `param_key` | text (PK) | Yes | Unique parameter identifier |
| `param_value` | text | Yes | Stored value (interpreted based on `param_type`) |
| `param_type` | param_type (enum) | Yes | Value type: `integer`, `decimal`, `boolean`, `text`, or `email_list` |
| `description_nl` | text | No | Dutch description of the parameter |
| `description_fr` | text | No | French description of the parameter |
| `created_on` | timestamptz | Yes | Record creation timestamp |
| `created_by` | uuid | No | User who created the record |
| `updated_on` | timestamptz | Yes | Last update timestamp |
| `updated_by` | uuid | No | User who last updated the record |

<Callout kind="tip">
  The `parameter` table uses `param_key` (text) as its primary key rather than an auto-incrementing integer. This makes it easy to reference parameters by name in application code (e.g., `auto_logout_minutes`, `notification_recipients`).
</Callout>

<Expandable title="SQL definition" default-open="false">
```sql
create type param_type as enum ('integer', 'decimal', 'boolean', 'text', 'email_list');

create table parameter (
  param_key      text primary key,
  param_value    text not null,
  param_type     param_type not null,
  description_nl text,
  description_fr text,

  created_on timestamptz not null default now(),
  created_by uuid references auth.users(id),
  updated_on timestamptz not null default now(),
  updated_by uuid references auth.users(id)
);
```
</Expandable>

## notification_log

Audit trail for email notifications sent by the system. Uses a polymorphic reference pattern (`entity_type` + `entity_id`) to link notifications to any entity.

| Column | Type | Required | Description |
|--------|------|----------|-------------|
| `notification_id` | bigint (identity PK) | Yes | Auto-generated unique identifier |
| `entity_type` | text | Yes | Type of related entity (e.g., `contract`, `invoice`) |
| `entity_id` | text | Yes | Identifier of the related entity |
| `notification_type` | text | Yes | Category of notification (e.g., `inspection_expiry`, `invoice_sent`) |
| `recipients` | text | Yes | Email recipients |
| `subject` | text | No | Email subject line |
| `sent_on` | timestamptz | Yes | When the notification was sent |
| `status` | text | Yes | Delivery status (default: `sent`) |
| `created_on` | timestamptz | Yes | Record creation timestamp |
| `created_by` | uuid | No | User who created the record |
| `updated_on` | timestamptz | Yes | Last update timestamp |
| `updated_by` | uuid | No | User who last updated the record |

**Indexes:**

| Index | Columns | Purpose |
|-------|---------|---------|
| `idx_notification_log_entity` | `entity_type, entity_id` | Fast lookup by related entity |

<Expandable title="SQL definition" default-open="false">
```sql
create table notification_log (
  notification_id bigint generated always as identity primary key,
  entity_type     text not null,
  entity_id       text not null,
  notification_type text not null,
  recipients      text not null,
  subject         text,
  sent_on         timestamptz not null default now(),
  status          text not null default 'sent',

  created_on timestamptz not null default now(),
  created_by uuid references auth.users(id),
  updated_on timestamptz not null default now(),
  updated_by uuid references auth.users(id)
);

create index idx_notification_log_entity on notification_log(entity_type, entity_id);
```
</Expandable>
