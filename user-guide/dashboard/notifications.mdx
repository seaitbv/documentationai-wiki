The notification panel occupies the left side of the [dashboard](/user-guide/dashboard/overview). It displays a priority-sorted list of action items that require your attention. Notifications are computed in real time from your data -- there is no separate notification inbox or table.

## How notifications work

- Notifications are **generated automatically** each time you load the dashboard.
- They are **sorted by priority** (highest first), then alphabetically within the same priority level.
- Each notification is **clickable** and navigates you to the relevant detail screen.
- You **cannot dismiss** notifications manually. They disappear automatically when the underlying issue is resolved.

<Callout kind="info">
  Because notifications are data-driven, resolving the underlying issue (e.g., renewing an expired inspection) immediately removes the notification on your next dashboard load.
</Callout>

## Notification types

### Inspection expired

| Property | Value |
|----------|-------|
| **Priority** | Critical |
| **Color** | Red |
| **Trigger** | Trailer inspection `valid_to` date is before today |
| **Navigates to** | Trailer detail screen (Fleet module) |

This is the highest-priority notification. It indicates that a trailer's technical inspection has expired. The trailer cannot legally be rented until the inspection is renewed.

<Callout kind="danger">
  Expired inspections represent a compliance risk. Resolve these notifications immediately by scheduling a new inspection for the affected trailer.
</Callout>

### Inspection expiring soon

| Property | Value |
|----------|-------|
| **Priority** | Attention |
| **Color** | Orange |
| **Trigger** | Trailer inspection `valid_to` is within the `INSPECTION_WARNING_DAYS` threshold |
| **Navigates to** | Trailer detail screen (Fleet module) |

This notification warns you that a trailer's inspection is approaching its expiry date. The default warning threshold is 14 days. Your administrator can adjust this through the [INSPECTION_WARNING_DAYS parameter](/user-guide/administration/parameters).

### Offer due date passed

| Property | Value |
|----------|-------|
| **Priority** | Attention |
| **Color** | Orange |
| **Trigger** | Open offer (status Created or Sent) has a `due_date` before today |
| **Navigates to** | Offer detail screen |

This notification indicates that an offer has passed its due date without being accepted or declined by the customer. Follow up with the customer or update the offer status.

### Contract sent without response

| Property | Value |
|----------|-------|
| **Priority** | Warning |
| **Color** | Yellow |
| **Trigger** | Contract has status "Sent" and was last updated more than 7 days ago |
| **Navigates to** | Contract detail screen |

This notification highlights contracts that were sent to the customer but have not received a response for over a week. Consider following up with the customer to confirm receipt.

### Missing km registration

| Property | Value |
|----------|-------|
| **Priority** | Info |
| **Color** | Blue |
| **Trigger** | Active kilometer-based contract has no km registration in the last 30 days |
| **Navigates to** | Contract detail screen |

This notification applies only to contracts that use kilometer-based billing. If no kilometer reading has been registered for more than 30 days, this reminder appears so you can update the records.

### Contract to check

| Property | Value |
|----------|-------|
| **Priority** | Attention |
| **Color** | Orange |
| **Trigger** | Contract has status "To check" (trailer not inspected after return) |
| **Navigates to** | Contract detail screen |

This notification indicates that a trailer has been returned but its condition has not been verified. Complete the post-return inspection and update the contract status.

## Priority reference

The following table summarizes all notification types sorted by priority (highest first).

| Priority | Color | Notification type | Typical action |
|----------|-------|-------------------|----------------|
| 1 - Critical | Red | Inspection expired | Schedule new inspection immediately |
| 2 - Attention | Orange | Inspection expiring soon | Plan inspection before expiry |
| 2 - Attention | Orange | Offer due date passed | Follow up with customer |
| 2 - Attention | Orange | Contract to check | Complete post-return inspection |
| 3 - Warning | Yellow | Contract sent without response | Contact customer for follow-up |
| 4 - Info | Blue | Missing km registration | Register current km reading |

<Callout kind="tip">
  Focus on red (critical) notifications first, then work through orange and yellow items. Blue notifications are informational and represent lower urgency.
</Callout>

## Resolving notifications

Each notification type has a specific resolution path:

<Steps>
  <Step title="Identify the notification type" icon="search" titleType="p">
    Check the color indicator and label to understand what kind of action is needed.
  </Step>

  <Step title="Click the notification" icon="mouse-pointer" titleType="p">
    Click the notification to navigate directly to the affected record's detail screen.
  </Step>

  <Step title="Resolve the underlying issue" icon="check-circle" titleType="p">
    Take the appropriate action based on the notification type:

    | Notification | Resolution |
    |-------------|------------|
    | Inspection expired/expiring | Add a new inspection record with updated dates |
    | Offer due date passed | Follow up with customer or update offer status |
    | Contract sent without response | Contact customer or update contract status |
    | Missing km registration | Add a new km registration entry |
    | Contract to check | Complete trailer inspection and update status |
  </Step>

  <Step title="Verify the notification is cleared" icon="eye" titleType="p">
    Return to the dashboard. The notification disappears once the underlying data condition is resolved.
  </Step>
</Steps>
