---
title: "Month-based invoicing"
description: "Understand how ARMS calculates month-based invoice proposals with pro-rata billing."
---

Month-based invoicing calculates billing amounts using pro-rata months. If a contract does not cover the entire month, ARMS calculates the fraction of the month used and bills accordingly.

## Proposal columns

The month-based proposal table displays:

| Column | Description |
|--------|-------------|
| **Contract ID** | Link to the contract detail |
| **Customer** | Customer name |
| **Trailer** | Trailer display name |
| **Line start** | First day of the month or the contract start date (whichever is later) |
| **Line end** | Last day of the month or the contract end date (whichever is earlier) |
| **Months (pro-rata)** | Fractional months used in this billing period |
| **Unit price (rental)** | The per-month rental price |
| **Discount (%)** | Discount percentage on the rental |
| **Unit price (insurance)** | The per-month insurance price |
| **Net total (excl. VAT)** | Calculated total before VAT |

## Calculation logic

<Steps>
  <Step title="Determine effective dates" icon="calendar" titleType="p">
    The system uses the **real dates** (pickup/dropoff) when available. Otherwise, it falls back to **estimated dates**.

    ```
    effective_start = real start date OR estimated start date
    effective_end   = real end date OR estimated end date
    ```
  </Step>

  <Step title="Calculate the billing period" icon="scissors" titleType="p">
    For the selected month, ARMS calculates the overlap with the contract:

    ```
    line_start = MAX(first day of month, effective start)
    line_end   = MIN(last day of month, effective end)
    ```

    If line start is after line end, the contract does not appear in the proposals.
  </Step>

  <Step title="Calculate pro-rata months" icon="calculator" titleType="p">
    ```
    days_used     = (line_end - line_start) + 1
    days_in_month = total days in the selected month
    months_fraction = CEILING(days_used / days_in_month, 2 decimals)
    ```

    The fraction is rounded **up** to 2 decimal places.
  </Step>
</Steps>

<Callout kind="info">
  Non-driving days do **not** apply to month-based invoicing. Only day-based contracts use non-driving day deductions.
</Callout>

## Pro-rata example

A contract starting on February 26, 2026, billed for February 2026:

| Value | Calculation |
|-------|-------------|
| Days used | Feb 26, 27, 28 = **3 days** |
| Days in February 2026 | **28 days** |
| Pro-rata fraction | 3 / 28 = 0.1071... rounded up = **0.11 months** |

## Generated invoice lines

ARMS generates two invoice lines for month-based contracts:

| Line | Quantity | Unit price | VAT | Discount applies? |
|------|----------|------------|-----|-------------------|
| **Trailer rental -- [month/year]** | Pro-rata months | Rental unit price | Customer VAT % | Yes |
| **Insurance -- [month/year]** | Pro-rata months | Insurance unit price | 0% | No |

<Callout kind="alert">
  The discount percentage applies **only to the rental line**, never to insurance. Insurance is always invoiced at the full unit price with **0% VAT**.
</Callout>

## Example calculation

A contract with the following details:
- Billing month: March 2026 (31 days)
- Contract period: March 10 - March 31 (22 days)
- Unit price rental: 1,500 EUR/month
- Unit price insurance: 200 EUR/month
- Discount: 5%
- Customer VAT: 21%

**Pro-rata months**: 22 / 31 = 0.7096... rounded up = **0.71**

| Line | Qty | Price | Discount | VAT | Total |
|------|-----|-------|----------|-----|-------|
| Trailer rental -- Mar 2026 | 0.71 | 1,500.00 | 5% | 21% | 1,011.75 |
| Insurance -- Mar 2026 | 0.71 | 200.00 | -- | 0% | 142.00 |
| | | | | **Subtotal** | **1,153.75** |
